# Advantages

## Hedera-Native Data

The pipeline starts from Hedera Mirror Node APIs instead of generic exchange-only data.

## Low-Friction Access

Mirror Node reads do not require API keys for common public endpoints, which makes demos and local experiments easier to reproduce.

## Agent-Ready Boundaries

Data fetch, metric aggregation, position sizing, and risk checks are separated so agent systems can use only the pieces they need.

## Honest ML Posture

The project documents leakage risks and treats execution quality, risk controls, and measurement as first-class concerns.

## Path To Larger Agent Systems

This package can stand alone or serve as the clean public data/risk layer for Hedera-focused tools that may be backed by in-house QVX/Veda infrastructure.
